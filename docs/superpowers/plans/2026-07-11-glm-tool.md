# GLM Tool Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose the user's Z.AI monthly GLM subscription as a quota-gated `ask_glm` tool for the local main model.

**Architecture:** CodexBar remains the sole quota source through provider id `zai`. LiteLLM receives one `glm-cloud` alias matching the working CCS profile (`https://api.z.ai/api/anthropic`, `glm-5.2`). Daily Assistant binds `ask_glm`; the tool checks CodexBar quota before calling LiteLLM and fails local when quota is unknown or exhausted.

**Tech Stack:** Python 3, httpx, pytest, LiteLLM YAML, SOPS runtime env files, shell verifiers.

## Global Constraints

- All LLM inference goes through LiteLLM at `http://127.0.0.1:4000`.
- Never commit the supplied API key or print it in tests/logs.
- GLM is a tool, never the default model or automatic fallback.
- `Z_AI_API_KEY` is the single environment-variable name shared with CodexBar.
- Missing credentials, quota, or provider health keeps the request local.
- No new dependency; reuse installed `httpx`, CodexBar models, and `ChatLiteLLM`.
- Static and both render targets must pass; live enablement still requires hardware verification.

class ZaiProvider(BaseProvider):
    def __init__(self, api_key=None, config=None):
        super().__init__("zai", api_key, _quota_url(), config)

    async def fetch_usage(self):
        async with httpx.AsyncClient(timeout=10) as client:
            response = await client.get(self.base_url, headers={
                "Authorization": f"Bearer {self.api_key}", "Accept": "application/json"
            })
        response.raise_for_status()
        return self.parse_usage(response.json())
```

`parse_usage()` maps `TOKENS_LIMIT` to primary, `TIME_LIMIT` to secondary, percent values to `0..1`, unit `1/3/5` to days/hours/minutes, and epoch-ms `nextResetTime` to UTC.

@tool
def ask_glm(prompt: str) -> dict:
    """Ask GLM only when local judgment needs a second cloud model; never send secrets/private context."""
    quota = _zai_usage()
    if not _zai_quota_available(quota):
        return {"status": "local_only", "tool": "ask_glm", "detail": "Z.AI quota unavailable"}
    try:
        return {"status": "ok", "tool": "ask_glm", "content": _invoke_glm(prompt)}
    except Exception as exc:
        return {"status": "error", "tool": "ask_glm", "detail": str(exc)}
```

`_invoke_glm()` uses `ChatLiteLLM(model="glm-cloud", api_base=LITELLM_BASE_URL, custom_llm_provider="openai", api_key="aipc-local", max_retries=0)`; add `ask_glm` to `TOOLS` and its use constraints to `SYSTEM_PROMPT`. Do not add it to `try_direct_tool()`.
